Claude API：构建 LLM 应用的官方指南

📦 anthropics/skills ⭐ 92.1k 🏷️ Data & AI

Claude API Skill 是 Anthropic 官方提供的开发指南，帮助你使用 Claude API 或 Agent SDK 构建 LLM 应用。当代码中出现 import anthropic 或用户询问 Claude API 时自动激活。

💡 核心价值：自动检测项目语言（Python/TypeScript/Java/Go/Ruby/C#/PHP），提供对应的代码示例。默认使用 Claude Opus 4.6 + 自适应思考 + 流式传输。

技能亮点：

  • 语言自动检测 — 根据项目文件自动识别 Python/TS/Java 等
  • 表面选择指南 — 单次调用/工作流/Agent，推荐最简方案
  • Tool Runner 支持 — Beta 工具装饰器，自动处理循环
  • Agent SDK 集成 — 内置文件/Web/终端工具和权限
# 安装
npx skills add anthropics/skills/claude-api

# 使用场景
- 分类、摘要、提取、问答 → Claude API 单次调用
- 多步骤流水线 → Claude API + tool use  
- AI Agent 需要文件/Web/终端 → Agent SDK

Sessions：VSCode Agent Sessions 窗口架构

📦 microsoft/vscode ⭐ 182.7k 🏷️ Development

Sessions Skill 是 Microsoft VSCode 团队的内部开发指南，详细说明了 Agent Sessions 窗口的架构设计。如果你在开发 VSCode 扩展或想了解 AI 编辑器架构，这是必读材料。

💡 核心价值：理解 VSCode 如何从"编辑器优先"转变为"对话优先"的 UI 架构。Chat Bar 成为主要组件，编辑器变成模态叠加层。

架构要点：

  • 分层约束 — vs/sessions 可导入 vs/workbench，但反向禁止
  • 简化 Chrome — 无活动栏、状态栏、横幅，聚焦对话
  • 固定布局 — 不支持设置自定义，保持一致体验
  • 规范文档 — README.md、LAYOUT.md、AI_CUSTOMIZATIONS.md
# 规范文档位置

Prompt Lookup：AI Prompt 搜索与优化

📦 f/prompts.chat ⭐ 152.7k 🏷️ Data & AI

Prompt Lookup Skill 让你通过 MCP 协议搜索、获取和优化 AI Prompt。当用户需要 prompt 模板或想改进现有 prompt 时自动激活。

💡 核心价值：直接在编程助手中访问 prompts.chat 的海量 prompt 库，无需离开工作环境。支持按类型（TEXT/IMAGE/VIDEO）和分类筛选。

MCP 工具：

  • search_prompts — 关键词搜索，支持类型/分类/标签过滤
  • get_prompt — 获取特定 prompt，自动处理变量填充
  • improve_prompt — AI 增强 prompt 质量
# 触发场景
"找一个代码审查的 prompt"
"有什么写作相关的 prompt？"
"帮我优化这个 prompt"

# 搜索参数
query: 搜索关键词
limit: 结果数量（默认 10，最多 50）
type: TEXT | STRUCTURED | IMAGE | VIDEO | AUDIO
category: 分类 slug（如 coding, writing）

n8n Conventions：自动化工作流开发规范

📦 n8n-io/n8n ⭐ 179.3k 🏷️ Development

n8n Conventions Skill 是 n8n 自动化平台的开发速查卡。n8n 是 Zapier/Make 的开源替代品，这个 Skill 帮助你快速了解代码规范和最佳实践。

💡 核心价值：在 AI 编程助手中快速查阅 n8n 的 TypeScript 规范、错误处理、前后端架构。避免踩坑。

关键规则：

  • TypeScript — 禁止 any，用 unknown；优先 satisfies
  • 错误处理 — 使用 UnexpectedError，不用废弃的 ApplicationError
  • 前端 — Vue 3 Composition API，CSS 变量，i18n
  • 后端 — Controller → Service → Repository，依赖注入
  • 数据库 — 仅 SQLite/PostgreSQL（应用数据库）
# 错误处理示例
import { UnexpectedError } from 'n8n-workflow';
throw new UnexpectedError('message', { extra: { context } });

# 测试命令
pushd packages/cli && pnpm test

# 完整文档
/AGENTS.md                    # 架构、命令、工作流
/packages/frontend/AGENTS.md  # CSS 变量、动画时间

OpenAPI Regen：自动重生成 API 客户端

📦 Significant-Gravitas/AutoGPT ⭐ 182.5k 🏷️ Development

OpenAPI Regen Skill 来自 AutoGPT 项目，当后端 API 路由变化时自动重新生成 OpenAPI 规范和前端类型客户端。一键完成"启动后端 → 获取规范 → 生成 hooks → lint + format"的完整流程。

💡 核心价值：前后端类型同步的痛点一键解决。支持用户直接调用（user-invocable: true），不仅 AI 自动触发。

触发场景：

  • API 路由变更
  • 新增端点
  • 前端 API 类型过期
# 一键执行（单个 shell 块保持 PID）
cd autogpt_platform/backend && poetry run rest &
REST_PID=$!
WAIT=0; until curl -sf http://localhost:8006/health; do 
  sleep 1; WAIT=$((WAIT+1)); [ $WAIT -ge 60 ] && exit 1
done
cd ../frontend && pnpm generate:api:force
kill $REST_PID
pnpm types && pnpm lint && pnpm format

# 规则
- 始终用 pnpm generate:api:force
- 不要手动编辑 src/app/api/__generated__/
- 生成的 hooks 命名：use{Method}{Version}{OperationName}
📂 技能地址： github.com/Significant-Gravitas/AutoGPT
